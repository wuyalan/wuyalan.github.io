毕业前一年，来谈谈房子

评论

先说说自己的感受吧，十多年来房价的一路飙升，早可预见这匹脱缰的野马，在未来大概也鲜少会有所收敛，作为还没准备好去踏入社会的小青年，内心是充满恐惧的。说实话，现在房子这东西，不靠父母、不借外力，单凭自己的力量，刚毕业能在这方面去把准备做好完全就是个伪命题。

有人之前举过一个例子，一个普通的硕士研究生，2003年毕业后在北京找到一份工作，月薪4K，那时候的房价与他工资水平相当，每平4k左右；工作十年后这个人的工资增加了3倍，变成每月一万二，可是房价这时候已经上升到了四万，所以结论竟然是他越努力挣钱、反倒越买不起房。最近一线城市的房价又开始以及其诡异的姿态猛涨，还有学者描绘了一幅“诱人”的前景：说按现在每年房价上涨百分之十几的比率如此推算，二十年以后北京的房价大概会上升至每平米80万。人们也都是这样建议：“尽早买房”——在交通隘口的地方，买套房先驻扎下来，这个资产就会升值。

中国的房地产经济所呈现出的怪像，套在任何一个现有的经济学理论模型里都得不出结果。随着市场的越来越开放、资源越来越集中，北上广深杭这些核心城市的房价只会继续猛涨。而另一个现象是二三线的中小城市近些年房价上涨幅度并不明显，房地产经济已经出现分化，不是所有地方在升值。

马光远老师说：“全球经济已经进入了一个‘平庸的十年’”。过去二三十年，应该是人类有史以来经济最好的一个阶段。这段时间造成经济出现这种超级繁荣的因素有三：第一个叫弱势美元，第二个叫全球流动性过剩，第三个是包括中国在内的新兴市场突然崛起。但是现在这三个力量都没了——弱势美元被强势美元替代、全球流动性过剩被流动性短缺替代，第三个，新兴市场的超级崛起，不管你承不承认，都已经画上了句号。有人说，现在经济形势现在这么不景气，是因为中国在以前做了太多违背经济学原理的事情，到现在报应来了。那么让我们看看中国的房地产经济到底是怎样兴起的——

中国在1993年开始取消票据之前的近20年时间里，是短缺性经济。而在90年代之后，中国的经济政策的目标发生了微妙的变化：从为了解决生产力不足的经济复苏政策转移为对经济的集权化控制。然而从1997年夏季开始，美国的对冲基金狙击亚洲各国货币，引爆“东亚金融危机”，整个亚洲地区全部垮掉。于是，东南亚金融风暴演变为亚洲金融危机，金融风暴肆虐周边各国。伴随着金融危机，国内的工业库存还出现了“结构性过剩”，95%的工业品供大于求。更让人担忧的是，进入1998年，国内出现民营企业第一次大规模倒闭浪潮，中国出现自1988年之后的又一次大萧条。

面临着通货紧缩、产能过剩，朱镕基以三大经济政策，将中国经济强行拉出泥潭。98年里中国做出了三个产业变更，第一，逐渐取消外贸限制权：第一次出现了“中国制造”。第二，启动城市化建设：中国政府发布了六千亿国债，主要投资于基础设施，如高速公路、交通、发电和大型水利工程。修高速公路的时候，地方政府把土地从农民手上买过来卖给开发商。第三，刺激内需，开放房地产市场：1998年7月，中央政府取消了福利分房，全国普及按揭贷款。几乎同时，中国人民银行颁布《个人住房贷款管理办法》，允许商业银行开展住房按揭贷款的服务。这两大措施，直接刺激了房地产业的复苏，中国金融到了一个很长时间的房地产的黄金时期。中国的房价开始涨，老百姓把一辈子的钱拿出来买房子，买了房子以后要买所有的东西，带动了和房地产相关的60多个行业。

对于繁荣了这么多年的房地产经济，那些持所谓的“政治阴谋论”者也不在少数。有人分析说，其实对于统治者，最核心的问题就是必须保证大部分年轻人处在贫穷困苦又充满希望的状态。对于一个国家来说，高房价就是维持统治的绝佳手段——日本的80年代，法国的后戴高乐时代，英国的撒切尔夫人时代，美国的里根年代，都是通过房价暴涨稳定了蠢蠢欲动的年轻人去好好干活，维持了社会的生产力。对于一个城市而言，高房价其实起到一个驱逐作用，只有最优秀的人才能留下来，其他的人都要慢慢移除掉，这对于城市的发展是很有利的。

而身处于跌宕中的我们，纵使能够宏观到其缘由为何，也逃离不出这场风暴圈，该过的生活还是依旧。至于买房？留在大城里凑够了钱、赶紧买来坐等升值的，和一路打拼、刷够了Buff后再移居小城的，都挺安然快活，个人有个人的活法，找到自己舒服那个方式就好。而毕业前，最好应得是去找准一个工资涨幅还勉强能跟上房价的职业，比如码农一类，好好学习、努力奋斗，大抵还能在大城市里混到个糊口罢。

用Github搭建个人独立博客——小白篇

Hexo, 技术帖

本人技术小白，为了搭个自己的独立博客，撒欢地跑去百度了一堆教程，结果——为神马明明是用同样的工具但这些教程都写得不一样啊喂！为神马明明跟着一步步做结果还是出错啊喂！程序员大哥你是不是省略了好多步骤……所以，我决定要将我搭建博客的过程全记录下来，以供后期和我一样的小白参考（是的，我坚信还有很多和我一样的人），其中的资料可能会摘录到其他人的教程，会在后面列出参考资料，感谢这些作者们。

什么是Github

这张图实在太经典了[捂脸]，1秒让你掌握Github作用

快速开始

安装准备软件

依次下载安装:

Node.js

Git

【提示】不要安装最新版本的Git！贴主在Git官网下载安装了最新版本后发生了一系列不可预估的错误……

注册GitHub

访问：http://www.github.com/ ，注册你的 username 和邮箱。uername 最好都用小写，因为最后建立的博客地址是：username.github.io；邮箱十分重要，GitHub 上很多通知都是通过邮箱的。

创建Repository

在自己 Github 主页的右上角，创建一个新的 Repository。

创建的 Repository 如下图。比如贴主的 Github 账号是 wuyalan，那么创建的 Repository 名字应该是 wuyalan.github.io。

配置和使用Github

打开Git Bash

配置SSH keys

怎样让本地 git 项目与远程的 github 建立联系咧？这里就需要配置好 SSH keys。

1. 检查SSH keys的设置

首先要检查自己电脑上现有的 SSH key：

$ cd ~/. ssh

如果显示“No such file or directory”，说明这是你第一次使用 git，跳到第三步；

如果没有显示上面这句话，继续看第2步。

2. 备份和移除原来的 SSH key设置

因为已经存在 key 文件，所以需要备份旧的数据并删除：

$ ls
config id_rsa id_rsa.pub known_hosts
$ mkdir key_backup
$ cp id_rsa* key_backup
$ rm id_rsa*

3. 生成新的SSH Key

输入下面的代码，就可以生成新的 key 文件啦。

【提示1】这里的邮箱地址，输入注册 Github 的邮箱地址；

【提示2】「-C」的是大写的「C」。

$ ssh-keygen -t rsa -C "邮件地址@youremail.com"
Generating public/private rsa key pair.
Enter file in which to save the key (/Users/your_user_directory/.ssh/id_rsa):<回车就好>

然后系统会要求设置一个密码：

Enter passphrase (empty for no passphrase):<设置密码>
Enter same passphrase again:<再次输入密码>

以后通过 Github 上传文件到 Repository 时会用到这个密码；如果不填密码就直接回车的话，以后上传时也就不会有密码验证啦。当然这个设置是防止别人往你自己的 Repository 里提交内容哒。
【提示】输入密码的时候什么都不会显示！不要以为没输进去，输入完直接回车就可以了。
最后看到这样的界面，就表示已经成功设置好了 SSH key：

添加SSH Key到GitHub

在本机设置 SSH Key 之后，需要添加到 GitHub 上，以完成 SSH 链接的设置。

1、在我的电脑打开 C:\Users\Administrator.ssh 文件夹，找到 id_rsa.pub 文件，这里面就是刚才生成的密钥。用记事本打开，全选并复制。如果看不到这个文件，就需要通过下图的步奏来设置显示隐藏文件。准确的复制这个文件的内容，才能保证设置的成功。

2、登陆 github 系统。点击右上角的 Edit profile -> SSH keys -> new SSH keys，把刚刚生成的密钥复制到里面（key文本框中），点击 add key 就可以啦。

测试

可以输入下面的命令，看看设置是否成功，git@github.com 的部分不需要修改：

$ ssh -T git@github.com

如果是下面的反馈：

The authenticity of host 'github.com (207.97.227.239)' can't be established.
RSA key fingerprint is 16:27:ac:a5:76:28:2d:36:63:1b:56:4d:eb:df:a6:48.
Are you sure you want to continue connecting (yes/no)?

不要紧张！输入 yes、回车，接下来你需要输入刚刚设置好的密码，再回车。

Enter passphrase for key '/c/Users/Administrator/.ssh/id_rsa':
Hi wuyalan! You've successfully authenticated, but GitHub does not provide shell access.

设置用户信息

现在已经可以通过 SSH 链接到 GitHub 啦!当然还需要完善一些个人信息:

$ git config --global user.name "wuyalan"//输入注册时的username
$ git config --global user.email  "alan.wyl@foxmail.com"//填写注册邮箱

SSH Key配置成功

本机已成功连接到 github。

如有问题，请重新设置。常见错误请参考：

GitHub Help - Generating SSH Keys

GitHub Help - Error Permission denied (publickey)

搭建hexo博客

利用npm命令安装hexo

$ cd
$ npm install -g hexo

搭建本地博客

1. 创建独立博客项目文件夹

安装完成后，关掉前面那个 Git Bash 窗口。在本地创建一个与 Repository 中博客项目同名的文件夹（如H:\username.github.io）在文件夹上点击鼠标右键，选择 Git bash；

【提示】在进行博客搭建工作时，每次使用命令都要在 H:\username.github.io 目录下。
执行下面的指令，Hexo 就会自动在 H:\username.github.io 文件夹建立独立博客所需要的所有文件啦！

$ hexo init

2. 安装依赖包

$ npm install

3. 确保git部署

$ npm install hexo-deployer-git --save

4. 本地查看

现在已经搭建好本地的 Hexo 博客了，执行完下面的命令就可以到浏览器输入 localhost:4000 查看到啦。

$ hexo g
$ hexo s

5. 用Hexo克隆主题

执行完 hexo init 命令后会给一个默认的主题：landscape，不过萝卜白菜各有所爱啦~你可以到 Hexo 的主题页 Hexo Themes 上去挑选自己喜欢的主题~找到它所在的 Github Repository，就可以通过 git clone 命令下载下来：

$ git clone https://github.com/daisygao/hexo-themes-cover.git themes/cover

贴主下载的是 Daisy Gao 的主题 cover。

6. 修改整站配置文件

【提示】为避免出错，先备份好初始的_config.yml 文件。

安装完成后，用记事本打开 username.github.io_config.yml 文件，全选、删除，将内容全部替换成下面的代码，并在带“#”的中文部分上进行相应的个性化修改：

# Hexo Configuration
## Docs: http://zespia.tw/hexo/docs/configure.html
## Source: https://github.com/tommy351/hexo/

# Site 这里的配置，哪项配置反映在哪里，可以参考我的博客
title: My Blog #博客名
subtitle: to be continued... #副标题
description: My blog #给搜索引擎看的，对网站的描述，可以自定义
author: Yourname #作者，在博客底部可以看到
email: yourname@yourmail.com #你的联系邮箱
language: zh-CN #中文。如果不填则默认英文

# URL #这项暂不配置，绑定域名后，欲创建sitemap.xml需要配置该项
## If your site is put in a subdirectory, set url as 'http://yoursite.com/child' and root as '/child/'
url: http://yoursite.com
root: /
permalink: :year/:month/:day/:title/
tag_dir: tags
archive_dir: archives
category_dir: categories

# Writing 文章布局、写作格式的定义，不修改
new_post_name: :title.md # File name of new posts
default_layout: post
auto_spacing: false # Add spaces between asian characters and western characters
titlecase: false # Transform title into titlecase
max_open_file: 100
filename_case: 0
highlight:
  enable: true
  backtick_code_block: true
  line_number: true
  tab_replace:

# Category & Tag
default_category: uncategorized
category_map:
tag_map:

# Archives 默认值为2，这里都修改为1，相应页面就只会列出标题，而非全文
## 2: Enable pagination
## 1: Disable pagination
## 0: Fully Disable
archive: 1
category: 1
tag: 1

# Server 不修改
## Hexo uses Connect as a server
## You can customize the logger format as defined in
## http://www.senchalabs.org/connect/logger.html
port: 4000
logger: false
logger_format:

# Date / Time format 日期格式，不修改
## Hexo uses Moment.js to parse and display date
## You can customize the date format as defined in
## http://momentjs.com/docs/#/displaying/format/
date_format: MMM D YYYY
time_format: H:mm:ss

# Pagination 每页显示文章数，可以自定义，贴主将10改成了5
## Set per_page to 0 to disable pagination
per_page: 5
pagination_dir: page

# Disqus Disqus插件，我们会替换成“多说”，不修改
disqus_shortname:

# Extensions 这里配置站点所用主题和插件，暂时默认
## Plugins: https://github.com/tommy351/hexo/wiki/Plugins
## Themes: https://github.com/tommy351/hexo/wiki/Themes
theme: landscape
exclude_generator:
plugins:
- hexo-generator-feed
- hexo-generator-sitemap

# Deployment 站点部署到github要配置
## Docs: http://zespia.tw/hexo/docs/deploy.html
deploy:
  type: git
  repository: 
  branch: master

7. 启用新下载的主题

在刚打开的的_config.yml 文件中，找到“# Extensions”，把默认主题 landscape 修改为刚刚下载下来的 cover：

【提示】username.github.io 里有两个 config.yml 文件，一个在根目录，一个在 theme 下，现在修改的是在根目录下的。

theme: cover

8. 更新主题

$ cd themes/cover
$ git pull

9. 本地查看调试

每次变更_config.yml 文件或者上传文件后都可以先用下面的命令调试一次，然后就可以在浏览器输入 http://localhost:4000，来查看博客搭建效果。

$ hexo g #生成
$ hexo s #启动本地服务，进行文章预览调试，退出服务用Ctrl+c

将博客部署到username.github.io

1. 复制SSH码

进入 Github 个人主页中的 Repository，复制新建的独立博客项目:username.github.io 的 SSH 码:

git@github.com:username/username.github.io.git

2. 编辑整站配置文件

打开 H:\username.github.io_config.yml,把刚刚复制的 SSH 码粘贴到“repository：”后面，别忘了冒号后要空一格。

deploy:
  type: git
  repository: git@github.com:username/username.github.io.git
  branch: master

3. 执行下列指令即可完成部署。

【提示】每次修改本地文件后，需要 hexo g 才能保存。每次使用命令时，都要在 H:\username.github.io 目录下。

$ hexo g
$ hexo d

或者直接作用组合命令

$ hexo d -g

【提示】如果在配置 SSH key 时设置了密码，执行 hexo d 命令上传文件时需要输入密码进行确认。

哈哈哈哈哈哈——现在博客已经完全搭建好了，在浏览器访问 username.github.io 就能看到啦！

写博文

好哒！既然博客站点已经配置好啦，那么贴主现在想要把自己的经验写到博客里发布出来，要怎么办咧？

1. 创建markdown文件

首先还是在 H:\username.github.io 目录下右键 Git Bash，执行 hexo n 命令，会生成指定名称的 markdown 文件至 username.github.io\source_posts\postName.md：

$ hexo new [layout] "postName"

其中“layout”是可选参数，默认值为 post，在命令中可以不写出来。有哪些 layout 呢，请到 scaffolds 目录下查看，这些文件名称就是 layout 名称。高级一点的呢，可以添加自己的 layout，方法就是添加一个文件即可，同时还可以编辑现有的 layout。

postName 是 md 文件的名字，同时也出现在自己文章的链接中，postName 如果包含空格，必须用”将其包围，postName 可以写成中文唷。

2. 编辑

在 H:\username.github.io\source_posts 中打开这个文件（打开方式用“记事本”就行，你也可以像贴主一样用小书匠等等的 markdown 编辑器来打开它），修改开头：

title: postName #文章页面上的显示名称，可以任意修改，不会出现在URL中
date: 2013-12-02 15:30:16 #文章生成时间，一般不改，当然也可以任意修改
categories: #文章分类目录，可以为空，注意:后面有个空格
tags: #文章标签，可空，多标签请用格式[tag1,tag2,tag3]，别忘了冒号后面有个空格
---
#这里开始使用markdown格式输入你的正文。

3. 用Markdown语法书写正文

接下来，就可以用喜爱的编辑器尽情书写博文啦。关于 markdown 语法，可以参考轻松搞定 Markdown

书友会里的“哲思详谈”

评论

［作为还没步入社会的学生党，看法或许过于片面、甚至脱离实际，记下此次活动的些许感想po出来，不足之处还望各位批评指正。］

不知是幸运还是不幸，这些年，创业似乎变得很容易。自入大学以来，耳边一直充斥着创业的声音。随便有点想法的学生，说”我要创业”，学校就会主动提供资金、提供培训、提供平台。高校的确提供了很好的资源大力扶持学生群体创业，同时这也使得现在的校园里出现了一大群还没学好基础知识、没在社会站稳脚跟、没有任何实际市场分析调研基础而仅仅是喜欢空想、看着遍布在新闻报刊的各种”创业神话”脑子一热突然就说自己要创业的人。大学里在外面跑过很多活动，总会遇到那么一群人，第一次见面，交流还没几句就说自己在创业、自己的想法是多么多么好、你要不要加入我们之类——听上去怎么这么像传销？首先，为什么会这么盲目自信自己的点子是最好的，有没有事先考察过市场、有没有详细的商业计划书；其次，面对一个才认识没多久的人就开始拉入伙，组建创始团队这样草率，你怎么知道我一定胜任这个职位，你怎么知道我不会是个坑？看到太多团队，一开始吹得自己多牛多牛前景多么多么好，拉到了投资把钱耗完后就不了了之了，创业大军里撑过A轮的太少太少。幸好是大学生，还年轻，输得起。哲哥和老刘说得对，把创业当做大学里的一种尝试和历练，在入职后用创业的心态做事，对于大学生的个人发展是有很大帮助的。

“创业维艰”，老刘说，很多人只看到了那些成功案例的表面，而忽视了它们背后倒下的千军万马，也忽视了那些成功在之前所累积的各种条件——马云背后的雅虎和高盛、雷军走出的金山、爱奇艺和百度、芒果TV和湖南广电，还有现在在长沙创始不到一年、关注度已经超越了很多本地老平台的P8，创始者是远大总裁之子张贤铭。老刘对有创业想法的大学生的建议很中肯，建议先入职工作，了解公司运营模式、累积人脉、接触行业最新的发展方向。

社会发展节奏越来越快，整个群体也变得越来越浮躁，那些“成功学”的故事让人们像打了鸡血一样，坐在办公室里天天觉得自己怀才不遇，静下心来认真做事的人太少。哲哥对刚入职场的新人的建议个人觉得很有帮助：发自内心地去主动学习，拓展行业相关知识；工作更重要的是看效率而不是加班时间；明白自己读研深造是为了什么：要培养什么方面的能力、以后要进入什么行业、要接触什么方面的人，而不仅仅是看一纸文凭；拓展人脉圈，认识那些给你未来职业带来帮助的人；把时间投资给能为你带来80％回报的人和事物上。

在学校和社会针对大学生的公益讲座里，总会宣扬很多关于领导力、关于口才、关于管理、关于人脉等软实力的东西，因为这些是光靠读书所不能获得的能力。但自己的观点还是认为：所谓的“软实力”，都是在“硬本领”基础上的“加码”，这个世界上仅仅凭借软实力成功的人太少太少。加入社团、做分兼职，但同时别忘了，好好学习，好好读书。

致我亲爱的20岁

随笔

即便知道这是逃不开的，还一直自欺欺人拖到靠后一点的农历，过完公历生日之后一直处于打死都不承认、他人一经提起就来火的状态——是的，我20岁了。

话出口，一个人就似孤单地站在海边，被怅然若失的情绪卷走，脑子里充斥的全是些奇奇怪怪的东西——什么“《我是歌手》再也不能蹭进10组了…”“啊就要奔三了我还没有好好享受童年…”“皮肤会老化以后不能肆无忌惮地熬夜嗨玩要开始系统地保养了…”我不是在怅惘自己的前20年什么也没干，相反我做的可能要溢满。但是时间，真的走得太快了啊，当还在人群中肆意地喧闹、还沉浸在迷醉和狂乱中、不断地催眠着自己“一切还早、我还年轻”之时，『叮』的一声，世界突然冷寂下来——到点了。

『我还有多少个20年。』

自己还有好多事情没有去做，自己还有好多东西需要去学。知道人生不会再重来，所以做过的事情绝对不能让自己后悔。所以想要体验到更多，所以想要在同样的时间里能看得更远，所以从高考自由以后跑各种有趣的活动、加入各种不同的组织，所以不遗余力地向外面介绍自己，想认识更多优秀的人、接触更多新鲜的思潮、补充更全面的自己…还不够，还不够。

爸爸总喜欢挂在嘴边的话：“人到什么年龄就该做什么事，该做完的任务终是要完成。”说太多次都催眠成了一句魔咒。然而自己并不是一个按部就班的人，只会做自己愿做的事，喜欢就激情满满，倦了便放手，如此洒脱。可是接下来的人生、20岁以后的世界，我还会是原来的我吗？

早就有征兆了。这一年，自己变了太多：从前每每冲到最前排占座的自己现在几乎天天迟到，从前即便很想睡还会用手撑着眼皮连连翻白眼听课的自己现在没有一节课认真听过，从前去图书馆只知道写作业的自己现在更多的是去看各种杂书，从前习惯于看过的摘录的自己现在会零零碎碎写一点东西，从前一直对代码很头痛的自己开始敲键盘，从前晚睡晚起的自己现在晚睡早起，从前只是随便跑跑的自己开始疯狂地健身，从前总是犯懒人症不愿好好保养和化妆的自己现在开始摆弄各种瓶瓶罐罐，从来很注意不在社交网络里留下私人信息痕迹的自己现在连头像都换成了本人，从前认识新朋友就会咋咋呼呼并会羡慕崇拜的自己现在只是近乎面瘫的一笑，从来被朋友痛骂的包子性格现在开始有了小心机，从前的自己在乎的是喜欢和有趣，现在只看到有没有用；越来越看重表面、越来越浮躁、越来越功利、越来越绿茶婊……忘却了曾经的渴望，忘掉了从前的幻想，好不容易找回了儿时的期盼，却发现如今的实力怕只会让其成为空想。据说人有四种状态“没能力但不自知”、“没能力却有自知”、“有能力有自知”和“有能力不自知”，处在第一种和第四种状态怕是最幸福的，但谁也不愿成为那第一种人。想要改变，想要变得更好，哪怕经历的蜕变会过于痛苦，只要不是现在的自己，我愿去承受。

致：我美丽的、可爱的、努力的、即将蜕变的、一去便不会复返的、20岁的自己。
